# Internationalization (i18n) Guide

## Overview

The app supports 6 languages with automatic RTL/LTR switching:

| Language | Code | Direction | Native Name |
|----------|------|-----------|-------------|
| English  | `en` | LTR       | English     |
| Russian  | `ru` | LTR       | Русский     |
| Georgian | `ka` | LTR       | ქართული     |
| Hebrew   | `he` | RTL       | עברית       |
| Arabic   | `ar` | RTL       | العربية      |
| Spanish  | `es` | LTR       | Español     |

## Architecture

```
src/lib/i18n/
├── translations.ts    # 6-language dictionaries (180+ keys each)
└── index.ts          # Zustand store + useT() hook
```

### Translation Store
- Uses Zustand with `persist` middleware
- Language preference stored in `localStorage` key `flight-monitor-lang-v1`
- `useT()` hook returns a translation function `t(key)`
- `useI18n()` hook for direct state access

### RTL/LTR Direction
- `LanguageDirSync` component updates `<html dir>` and `<html lang>` on language change
- Pre-paint script in `layout.tsx` sets dir/lang from localStorage before React hydration (prevents flash)

## Usage

### In Components
```tsx
import { useT } from "@/lib/i18n";

function MyComponent() {
  const t = useT();
  return <h1>{t("appTitle")}</h1>;
}
```

### Getting Current Language
```tsx
import { useI18n } from "@/lib/i18n";

function MyComponent() {
  const lang = useI18n((s) => s.lang);
  const setLang = useI18n((s) => s.setLang);
  // ...
}
```

### Language Switcher
```tsx
import { LanguageSwitcher } from "@/components/dashboard/LanguageSwitcher";

function Header() {
  return <LanguageSwitcher />;
}
```

## Translation Keys

All keys are defined in `src/lib/i18n/translations.ts`. The English dictionary is the source of truth — all other languages must have the same keys.

Key categories:
- **App**: `appTitle`, `appSubtitle`, `active`, `uptime`, `healthScore`
- **Tabs**: `tabOverview`, `tabTrackers`, `tabForecast`, `tabProviders`, `tabSystem`
- **Tracker**: `newTracker`, `route`, `from`, `to`, `dates`, `cabin`, `passengers`
- **Cabin**: `economy`, `premium`, `business`, `first`
- **Forecast**: `aiForecast`, `buyNow`, `wait`, `keepMonitoring`, `confidence`
- **Alerts**: `deal`, `targetHit`, `vsAverage`, `low`, `average`, `high`
- **System**: `cpu`, `ram`, `disk`, `dbSize`, `logs`, `errors`

## Adding a New Language

1. **Add language metadata** in `translations.ts`:
```ts
{ code: "fr", name: "Français", englishName: "French", flag: "🇫🇷", dir: "ltr" },
```

2. **Add translation dictionary**:
```ts
fr: {
  appTitle: "Agent de Surveillance des Vols",
  // ... all 180+ keys
},
```

3. **Add region labels** in `src/lib/airports.ts`:
```ts
NA: { en: "North America", ..., fr: "Amérique du Nord" },
```

4. **Add cabin labels** in `src/lib/priceEngine.ts`:
```ts
economy: { en: "Economy", ..., fr: "Économique" },
```

5. **Add forecast reasoning templates** in `src/app/api/forecast/route.ts` and `mini-services/forecast-service/index.py`.

## Date Formatting

Dates are formatted using the browser's `Intl.DateTimeFormat` with locale mapping:

```ts
const dateLocaleMap = {
  en: "en-US",
  ru: "ru-RU",
  ka: "ka-GE",
  he: "he-IL",
  ar: "ar",
  es: "es-ES",
};

function formatDateShort(dateStr: string, lang: string) {
  return new Date(dateStr).toLocaleDateString(dateLocaleMap[lang], {
    day: "numeric",
    month: "short",
  });
}
```

## RTL Considerations

- Charts use `dir="ltr"` to prevent axis label issues
- Log entries use `dir="auto"` for mixed-language content
- Flexbox layouts naturally flip in RTL
- Icons with directional meaning (arrows) may need swapping
